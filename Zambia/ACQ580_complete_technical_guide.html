1. How the ACQ580 works

The ACQ580 is a Variable Speed Drive (VSD) — also called a Variable Frequency Drive (VFD) — designed specifically for water and wastewater pump applications. It converts fixed-frequency mains AC power into variable-frequency, variable-voltage AC output to control motor speed and torque precisely.

Internal power conversion stages

StageWhat happensComponent
1. RectifierConverts 3-phase AC mains (230–480 V) to DC using a diode bridge rectifier4× 125J MOVs for surge protection
2. DC busStores energy in capacitor bank; provides stable DC link voltage (~1.35 × mains V)Electrolytic capacitors
3. IGBT inverterSwitches DC at high frequency (switching frequency 4–12 kHz) using PWM to recreate 3-phase AC at desired frequency (0–500 Hz)IGBT transistors
4. Control boardRuns the pump control firmware; handles PID, I/O, fieldbus, protectionsEmbedded microcontroller

Pump-specific features built in

  • PID loop (Process control): Takes feedback from a pressure/flow transducer on AI2; automatically adjusts motor speed to maintain setpoint
  • Anti-cavitation protection: Detects and reacts to pump cavitation by speed reduction
  • Dry-run protection: Stops the pump if it runs without water (via DI or current sensing)
  • Soft pipe fill: Ramps up slowly to prevent water hammer
  • Multi-pump (IPC): Controls up to 3 pumps in rotation; drive-to-drive communication via D2D
  • Sensorless flow calculation: Estimates flow from motor parameters without a flow meter
  • Safe Torque Off (STO): Hardware safety; built-in as standard

Operating modes

Local (Hand)

Controlled from the control panel (ACX-AP). Used during commissioning and maintenance. Panel overrides all external signals.

Remote (Auto)

Controlled via I/O terminals or fieldbus (Modbus RTU). Normal operating mode for pump station automation.

2. First-time setup & commissioning

⚠ Safety first: Only qualified electrical personnel should install and commission the drive. Dangerous voltages remain present for up to 5 minutes after power-off. Always verify the DC bus voltage is zero before working inside.

Step-by-step commissioning sequence

  1. Mechanical mounting: Mount vertically on a flat, vibration-free surface. Minimum clearance: 100 mm top & bottom, 50 mm sides. Operating temperature: −15 to +50 °C.
  2. Power wiring (mains input): Connect 3-phase supply (L1, L2, L3) to input terminals U1, V1, W1. Connect PE (protective earth) to the drive's PE terminal. Use copper conductors. Check mains voltage matches drive rating.
  3. Motor wiring (output): Connect motor to output terminals U2, V2, W2. Keep motor cable as short as possible; always use shielded motor cable and connect shield to PE at both ends.
  4. Control wiring: Connect I/O signals, transducer, and fieldbus to the control terminal block X2.
  5. Power up & Primary Settings wizard: On first power-up the ACX-AP control panel starts the Primary Settings wizard — set language, application type (Water default), motor nameplate data (voltage, current, frequency, speed, power).
  6. ID run: Perform motor ID run (stationary or rotating) via Menu → Primary settings → ID run to let the drive measure motor parameters automatically.
  7. Set control parameters: Set control source (EXT1), reference source, PID setpoint. Verify protections (overload, dry-run, cavitation thresholds).
  8. Test run: Start in Hand mode; verify rotation direction; check that the drive ramps up and down smoothly.

3. Modbus RTU configuration

The ACQ580 has a built-in Embedded Fieldbus (EFB) interface that operates as a Modbus RTU slave over RS485. No additional adapter card is needed for basic Modbus control and monitoring. For advanced protocols (PROFIBUS, EtherNet/IP, etc.) the optional FSCA-01 adapter can be used.

ℹ️ Protocol: The ACQ580 supports Modbus RTU (not ASCII). Physical interface: EIA-485 (RS485), half-duplex, 2-wire + signal ground.

Wiring for Modbus RTU

Cable recommendation: Use Belden 9842 or equivalent — dual twisted shielded pair, 120 Ω nominal impedance. Connect one twisted pair to B+ and A−. Use the second pair or a separate wire for DGND (signal ground).

Drive terminalConnect to master/gatewayNotes
X2:24 — B+RS485 B+ (Data+, non-inverting)Twisted pair, wire 1
X2:25 — A−RS485 A− (Data−, inverting)Twisted pair, wire 2
X2:26 — DGNDSignal GND on master/gateway3rd wire or 2nd pair — see Section 8
Termination switchSW ON if this drive is at bus end120 Ω internal termination
⚠ B+/A− vs A/B naming confusion: ABB labels their terminals B+ (non-inverting, logic high when idle) and A− (inverting). Some gateways label their terminals opposite. Always use a multimeter to verify polarity before first power-on of the bus. Reversed polarity causes no damage but communication will not work.

4. Modbus parameters & register map

Register address formula (ACQ580 / ACx580 series)

16-bit register: Modbus register = 40000 + (parameter group × 100) + parameter index
Example: Parameter 01.06 (Output frequency) → 40000 + (01×100) + 06 = 40106

32-bit register (param 58.33 = 0): 420000 + (200 × group) + (2 × index)
Example: Parameter 01.06 → 420000 + (200×1) + (2×6) = 420212

5. Reading data via USB–RS485 on a PC

✅ Yes, you can! A USB to RS485 converter connected to the drive's EFB port (B+, A−, DGND) lets you read and write parameters, monitor the drive, and even control it — without a PLC, gateway, or ABB Drive Composer software.

What you need

  • USB to RS485 converter — use a converter with a proper FT232 or CH340 chip, or better, a fully isolated converter (e.g. FTDI USB-RS485-WE, Waveshare USB to RS485, or similar)
  • Software: Any Modbus RTU master software — ModbusPoll (Windows), QModMaster (Linux/Win/Mac), modscan32, or a Python script using the pymodbus library
  • Wiring: Connect converter's A(−) to drive X2:25, B(+) to drive X2:24, GND to drive X2:26

PC software setup

SettingValue
PortCOM port assigned to your USB-RS485 adapter (check Device Manager)
ProtocolModbus RTU
Baud rateMatch drive P58.04 (default: 19200)
Data bits8
ParityEven (match P58.05, default: 8E1)
Stop bits1
Slave IDMatch drive P58.03 (default: 1)
Timeout500–1000 ms

Python example (read output frequency)

from pymodbus.client import ModbusSerialClient
client = ModbusSerialClient(port='COM3', baudrate=19200, parity='E', stopbits=1, bytesize=8, timeout=1)
client.connect()
result = client.read_holding_registers(address=105, count=1, slave=1) # Register 40106 → address=105
freq_raw = result.registers[0]
freq_hz = freq_raw / 20000 * 50 # if rated frequency is 50 Hz (P46.02)
print(f"Output frequency: {freq_hz:.2f} Hz")
client.close()
⚠ USB converter isolation: If using a non-isolated USB-RS485 converter, your PC ground will be referenced to the drive's DGND which is near the high-voltage power section. Always use a fully isolated USB-RS485 converter for safety and to prevent ground loop noise on the bus.

The most common sensor in a pump station is a 4–20 mA pressure transducer used for PID feedback (to maintain constant discharge pressure).

7. IoT gateway connection via Modbus RTU

An IoT/Modbus gateway (e.g. Advantech ADAM, Moxa, Ewon Cosy, Weintek, Robustel, or custom RS485→MQTT/Cloud gateway) acts as the Modbus master and the ACQ580 is the slave.

Connection topology

[IoT Gateway]
  RS485 B+ ──────── X2:24 B+ [ACQ580]
  RS485 A− ──────── X2:25 A− [ACQ580]
  GND ─────────── X2:26 DGND [ACQ580]
  (Termination 120Ω at gateway end if it is at the bus end)

Gateway configuration settings

  • Protocol: Modbus RTU (not TCP/ASCII)
  • Baud rate: match drive P58.04 (recommend 9600 for noisy environments, 19200 for default)
  • Parity: match drive P58.05 (default 8E1)
  • Slave address: match drive P58.03 (default 1)
  • Poll interval: minimum 100 ms; 500 ms–1 s is typical for pump monitoring
  • Timeout: 500–1000 ms
  • Register type: Holding Registers (function 03) for all ACQ580 parameters
  • Register addressing: 0-based (subtract 40001 from the Modbus register number) — e.g., register 40106 → address 105
⚠ 0-based vs 1-based addressing: Some gateways use 1-based addressing where "40106" is entered as-is. Others use 0-based where you enter 105. Check your gateway's manual. Entering the wrong offset is the #1 cause of "register not found" errors.

8. RS485 ground (DGND) connection — mandatory or optional?

⚠ DGND is mandatory for reliable communication in industrial environments with drives, motors, and switching equipment.

Why DGND matters

RS485 is a differential signalling standard — it can tolerate common-mode noise up to ±7 V. However, without a common ground reference between master and slave, ground potential differences can exceed this limit (especially near VSDs which generate significant common-mode voltage). This leads to corrupted data, CRC errors, and communication failures.

Correct DGND connection rules (per ABB ACQ580 manual)

  1. Connect the DGND conductor from drive X2:26 to the signal ground reference at the automation controller or gateway.
  2. Use a single DGND connection per device — do not create ground loops by connecting DGND at multiple points.
  3. The cable shield should be attached at each drive but NOT connected to the drive's PE/chassis ground — connect shields together at each drive via the SCR terminal, and connect the combined shield to the automation controller ground only at one end.
  4. Do NOT ground the RS485 network at more than one point along the bus (to prevent ground loop currents).

Termination resistors (120 Ω)

  • Required at both physical ends of the RS485 bus.
  • The ACQ580 has a built-in termination switch — set to ON for end-of-bus drives.
  • Only one device should have bias resistors enabled (pull-up/pull-down to define idle bus state) — typically the master/gateway end.
  • All intermediate drives: termination OFF, bias OFF.

9. EMI from the ACQ580 — effects & protection

⚠ The ACQ580 is a significant EMI source. IGBT switching at 4–12 kHz creates high-frequency electromagnetic interference that can corrupt RS485 communication, damage gateway circuits, and cause false readings in sensors.

EMI sources in the ACQ580

  • IGBT switching transients: Fast voltage rise/fall (dV/dt) on motor cable causes conducted and radiated EMI at harmonics of the switching frequency (4–12 kHz and harmonics)
  • Common-mode current: Flows through motor cable capacitance to PE, generating high-frequency ground currents that can contaminate signal cables sharing the same conduit
  • Motor cable radiation: Unshielded or poorly shielded motor cable acts as an antenna
  • DC bus switching noise: Visible on supply lines as conducted emissions

EMI compliance of ACQ580

The ACQ580 complies with EN 61800-3 (EMC product standard for drives). However, EMC compliance only holds under correct installation conditions — proper shielding, correct cable routing, and proper grounding.

How EMI affects the gateway and RS485 bus

  • Induced noise on RS485 cables → CRC errors, communication dropouts
  • Common-mode voltage transients → can damage non-isolated RS485 interfaces
  • High-frequency current through ground loops → corrupts signal reference
  • Physical proximity to motor cables → capacitive/inductive coupling

10. Non-opto-isolated RS485 in gateway — risks & precautions

⚠ If your IoT gateway does not have optically isolated RS485, the gateway's RS485 transceiver chip is directly referenced to the gateway's internal ground, creating a galvanic connection to the drive's DGND and all noise on that bus.

Risks of non-isolated RS485

  • Common-mode voltage damage: VSD-generated common-mode transients can exceed the RS485 chip's ±7 V tolerance, destroying the transceiver IC in the gateway — even a single surge can permanently damage it
  • Ground loop currents: Current flowing through the ground conductor between drive and gateway adds noise to data signals, causing CRC errors and communication dropouts
  • Data corruption: High-frequency switching noise from the VSD couples into the non-isolated RS485 line, corrupting frames
  • Gateway instability: Severe ground disturbances can lock up the gateway processor, requiring power cycle

Pros of non-isolated RS485

  • Lower cost gateway
  • Simpler implementation — no isolation barrier to manage
  • Acceptable in low-noise environments (e.g., office automation, small PLC panels far from large drives)

Precautions for non-isolated RS485 near ACQ580

#PrecautionWhy

  • Mount in the upper section of the panel or in a separate enclosure from the VSD
  • Use a metal DIN rail enclosure for the gateway (provides shielding)
  • Connect the enclosure to the PE/chassis ground star point
  • If in the same panel: use a grounded metal divider plate between VSD section and gateway/controls section

12. Complete pump station physical setup

A typical pump station using ACQ580 + pressure transducer + IoT gateway for cloud monitoring.

Critical wiring segregation rules for this setup

  1. Motor cable (U2,V2,W2): Always shielded; shield grounded at PE star point at both ends; run in its own conduit or at least 300 mm from signal cables
  2. Mains input (U1,V1,W1): Standard power cable in separate conduit from motor cable
  3. 4–20 mA transducer cable: Shielded, separate from motor cable, shield grounded at drive end only
  4. RS485 cable: Shielded, completely separate from motor cable, shield connected at one end only; install SPD at gateway entry
  5. Gateway power supply: From a clean 24 VDC SMPS, NOT from the drive's auxiliary output
